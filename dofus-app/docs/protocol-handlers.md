# Dofus 3.0 — Protocol handlers topology

Generated by `build-protocol-handlers.py` from a runtime IL2CPP scan.

**What this is**: a complete mapping `obfuscated_protocol_message → handler_class` extracted by scanning every method in `Core` whose parameter type is a Protobuf message class from `Ankama.Dofus.Protocol.Game`.

**What's still missing**: the original `.proto` names for each message (blocked at runtime — see `dofus-deobfuscation-roadmap.md` #1). The structural map below lets us cluster handlers by domain, and any future name recovery cascades through the graph.

## Re-using this map

Once we recover the original .proto name for any single message (e.g. via `FileDescriptor.FromGeneratedCode` invocation, or by hooking `MessageParser<T>.ParseFrom` at runtime when a real packet arrives), the cluster around it becomes labeled too. The JSON output `protocol-handlers.json` is the machine-readable source of truth, and `protocol-message-labels.json` lists the 17 messages already identified by clear-handler leak.
