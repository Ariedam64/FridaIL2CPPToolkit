# Dofus Plugin — Map Feature v1.1 (Cartography + Interactives + Neighbours) — Design Spec

> Iteration v1.1 du plugin Dofus map. Ajoute : (1) atlas Dofus stitché à la place des carrés colorés (le world canvas devient la vraie cartography in-game), (2) interactives drawn sur le cell-grid, (3) neighbours cliquables dans le panneau, (4) iso math fix landscape. Sub-project sur la même branche `feat/v1.5-dofus-map`.

**Date** : 2026-05-08
**Branche cible** : `feat/v1.5-dofus-map` (déjà créée pour v1)
**Auteur** : brainstormé avec l'utilisateur après testing manuel de v1
**Dépendances** : Dofus Map Feature v1 (commits `06c2a713..114beb02` sur cette branche, déjà mergés)

---

## TL;DR

Le user a testé v1 et signalé : (a) "plein de carrés colorés" — il voulait la vraie carte du jeu ; (b) cell-grid en portrait alors qu'in-game c'est landscape ; (c) pas d'interactables visibles ; (d) pas de navigation entre maps voisines.

v1.1 corrige ces 4 points. Le gros morceau est la **cartography** : on bundle 112 MB de tiles JPG depuis `dofus-app/data/cartography/`, on porte la logique d'atlas-rendering depuis `world.ts` (prior art sur `feat/dofus-app`), et le world canvas devient le vrai atlas Dofus stitché avec map markers en overlay semi-transparent.

Pas de zoom (scope v2). Pas de per-map preview PNG (8.9 GB, scope v2). Just le world atlas + interactives + neighbours nav + iso math fix.

**Out of scope v1.1** : zoom, pan, per-map PNG preview, resolution typeId→nom, tooltip hover sur interactives, auto-switch world dropdown sur cross-world neighbour, pre-fill placeholder grey avant tile load.

## Data model

### `tile-mapping.json` (copié verbatim depuis dofus-app)

```ts
type TileMappingFile = Record<string, MappedTile[]>;  // worldId → tiles list

interface MappedTile {
    index: number;       // sequential within (worldId, scale)
    name: string;        // worldId as string ("1" for Amakna)
    scale: string;       // zoom level identifier ("0.2" lowest, "0.4", ..., "1")
    address: string;     // game-side path "0.2/1.jpg"
    guid: string;        // Unity asset GUID (cross-ref with worldmap-addressables)
    tile: string | null; // actual filename in cartography/tiles/ (or null if missing)
    width: number;       // pixel width at native scale
    height: number;
    ambiguous: boolean;  // multiple game tiles share this address (rare)
}
```

Pour v1.1 on ne charge que `scale === "0.2"` (lowest, smallest footprint). Plus tard (zoom v2), pick `0.4` ou plus selon le zoom level visible.

interface WorldMapDims {

## Iso math fix (cell-grid)

### Bug actuel

`cell-grid.ts:25` :
```ts
const halfH = cellSize / 2;
canvas.width  = (COLS + 0.5) * cellSize;
canvas.height = (ROWS + 1) * halfH;   // = 41 * 8 = 328 with cellSize=16
```

Avec COLS=14 et ROWS=40, cellSize=16 → canvas 232 × 328 (portrait). Le rhombus dessiné via `cy ± halfH` est de 16 px haut, donc 1:1 avec sa largeur de 16 px → carré, pas iso.

### Fix

```ts
const halfV = cellSize / 4;   // half-height of the iso rhombus (cell ratio 2:1)
canvas.width  = (COLS + 0.5) * cellSize;
canvas.height = (ROWS + 1) * halfV;   // = 41 * 4 = 164 with cellSize=16
```

Et dans la boucle :
```ts
const cy = row * halfV + halfV;  // au lieu de row * halfH + halfH
ctx.moveTo(cx,             cy - halfV);  // au lieu de cy ± halfH
ctx.lineTo(cx + cellSize/2, cy);
ctx.lineTo(cx,             cy + halfV);
ctx.lineTo(cx - cellSize/2, cy);
```

Le rhombus devient 16 wide × 8 tall = ratio 2:1 (flat iso), comme dans `world.ts:106-117` du prior art. Canvas total : 232 × 164 — landscape ✓.

### Test update

`cell-grid.test.ts` — assertion height : `41 * 4 = 164` au lieu de `41 * 8 = 328`. Width inchangée.

## Atlas rendering (port from world.ts)

### `worldToAtlasXY`

```ts
function worldToAtlasXY(
    wm: WorldMapDims,
    posX: number,
    posY: number,
): { x: number; y: number; w: number; h: number } {
    return {
        x: (posX - wm.origineX) * wm.mapWidth,
        y: (posY - wm.origineY) * wm.mapHeight,
        w: wm.mapWidth,
        h: wm.mapHeight,
    };
}
```

### `atlasXYToWorld` (inverse, for hit-test)

```ts
function atlasXYToWorld(
    wm: WorldMapDims,
    atlasX: number,
    atlasY: number,
): { posX: number; posY: number } {
    return {
        posX: Math.floor(atlasX / wm.mapWidth)  + wm.origineX,
        posY: Math.floor(atlasY / wm.mapHeight) + wm.origineY,
    };
}
```

### Atlas tile flowing-row layout

```ts
const ATLAS_MAX_W = 3000;

async function renderAtlas(canvas, opts: { wm: WorldMapDims; tiles: MappedTile[]; maps: WorldMap[]; selectedMapId?: number|null; hoveredMapId?: number|null }): Promise<WorldCanvasResult> {
    const wm = opts.wm;
    const scale = Math.min(1, ATLAS_MAX_W / wm.totalWidth);
    canvas.width  = Math.round(wm.totalWidth  * scale);
    canvas.height = Math.round(wm.totalHeight * scale);

    const ctx = canvas.getContext("2d")!;
    ctx.fillStyle = "#0a0a0a"; ctx.fillRect(0, 0, canvas.width, canvas.height);
    ctx.imageSmoothingEnabled = true;

    // Filter to scale "0.2" only for v1.1
    const tilesForScale = opts.tiles.filter((t) => t.scale === "0.2");

    let cursorX = 0, cursorY = 0, rowMaxH = 0;
    for (const t of tilesForScale) {
        if (cursorX + t.width > wm.totalWidth + 8) {
            cursorY += rowMaxH; cursorX = 0; rowMaxH = 0;
        }
        const dx = cursorX * scale, dy = cursorY * scale;
        const dw = t.width * scale, dh = t.height * scale;
        if (t.tile) {
            try {
                const img = await loadTileImage(`/api/dofus/cartography/tile/${t.tile}`);
                ctx.drawImage(img, dx, dy, dw, dh);
            } catch {
                ctx.fillStyle = "#332"; ctx.fillRect(dx, dy, dw, dh);
            }
        } else {
            ctx.fillStyle = "#222"; ctx.fillRect(dx, dy, dw, dh);
        }
        cursorX += t.width;
        if (t.height > rowMaxH) rowMaxH = t.height;
    }

    // Map markers overlay (semi-transparent)
    const tileByXY: Record<string, number> = {};
    ctx.globalAlpha = 0.35;
    for (const m of opts.maps) {
        const a = worldToAtlasXY(wm, m.posX, m.posY);
        ctx.fillStyle = areaColor(m.areaId, m.mapId);
        ctx.fillRect(a.x * scale, a.y * scale, a.w * scale, a.h * scale);
        tileByXY[`${m.posX},${m.posY}`] = m.mapId;
    }
    ctx.globalAlpha = 1;

    // Selected map outline (full alpha)
    if (opts.selectedMapId != null) {
        const m = opts.maps.find((x) => x.mapId === opts.selectedMapId);
        if (m) {
            const a = worldToAtlasXY(wm, m.posX, m.posY);
            ctx.strokeStyle = "#fff"; ctx.lineWidth = 2;
            ctx.strokeRect(a.x * scale, a.y * scale, a.w * scale, a.h * scale);
        }
    }

    // Hovered map outline (semi-white)
    if (opts.hoveredMapId != null && opts.hoveredMapId !== opts.selectedMapId) {
        const m = opts.maps.find((x) => x.mapId === opts.hoveredMapId);
        if (m) {
            const a = worldToAtlasXY(wm, m.posX, m.posY);
            ctx.strokeStyle = "rgba(255,255,255,0.5)"; ctx.lineWidth = 1;
            ctx.strokeRect(a.x * scale, a.y * scale, a.w * scale, a.h * scale);
        }
    }

    const hitTest = (px: number, py: number): number | null => {
        const { posX, posY } = atlasXYToWorld(wm, px / scale, py / scale);
        return tileByXY[`${posX},${posY}`] ?? null;
    };
    return { hitTest };
}
```

### `loadTileImage` cache

```ts
const tileImageCache = new Map<string, HTMLImageElement>();

function loadTileImage(url: string): Promise<HTMLImageElement> {
    const cached = tileImageCache.get(url);
    if (cached) return Promise.resolve(cached);
    return new Promise((resolve, reject) => {
        const img = new Image();
        img.onload = () => { tileImageCache.set(url, img); resolve(img); };
        img.onerror = () => reject(new Error(`failed to load tile: ${url}`));
        img.src = url;
    });
}
```

Module-level cache. Pas d'éviction pour v1.1 (~1k tiles max ≈ <100 MB RAM).

## Interactives drawing (cell-grid)

Après la boucle de rendu des cells dans `cell-grid.ts`, ajouter :

```ts
if (opts.interactives) {
    for (const [cellIdx, _elementId, typeId] of opts.interactives) {
        const row = Math.floor(cellIdx / COLS);
        const col = cellIdx % COLS;
        if (row >= ROWS || col >= COLS) continue;  // out of bounds safety

        const cx = col * cellSize + (row & 1 ? cellSize / 2 : 0) + cellSize / 2;
        const cy = row * halfV + halfV;
        const radius = cellSize / 4;

        // Color hash on typeId (similar to areaColor pattern)
        const hue = (typeId * 137) % 360;
        ctx.fillStyle = `hsl(${hue}, 70%, 55%)`;
        ctx.strokeStyle = "#fff";
        ctx.lineWidth = 1;
        ctx.beginPath();
        ctx.arc(cx, cy, radius, 0, Math.PI * 2);
        ctx.fill();
        ctx.stroke();
    }
}
```

Visuellement : un petit rond coloré au centre de chaque cell d'interactive, contour blanc 1px pour la visibilité par-dessus n'importe quelle couleur de cell. Color stable pour un même typeId (le hash garantit que tous les Frênes ont la même teinte).

## Neighbours buttons (pages/map.ts)

Après le rendu du cell-grid dans le panneau side, ajouter un grid 3×3 de buttons (centre vide). Index dans `data.neighbours` = `[N=0, E=1, S=2, W=3]`.

```ts
function renderNeighbours(panel: HTMLDivElement, data: MapDetail, onClick: (mapId: number) => void): void {
    const [n, e, s, w] = data.neighbours.length === 4 ? data.neighbours : [0,0,0,0];
    const isValid = (id: number) => id !== 0 && id !== -1;

    const labelN = isValid(n) ? `↑ N (${data.posX}, ${data.posY - 1})` : "↑ N —";
    const labelE = isValid(e) ? `→ E (${data.posX + 1}, ${data.posY})` : "→ E —";
    const labelS = isValid(s) ? `↓ S (${data.posX}, ${data.posY + 1})` : "↓ S —";
    const labelW = isValid(w) ? `← W (${data.posX - 1}, ${data.posY})` : "← W —";

    const navHtml = `
        <div data-testid="neighbours-nav" style="display:grid;grid-template-columns:1fr 1fr 1fr;gap:4px;margin-top:12px;font-size:11px">
            <div></div>
            <button data-dir="n" data-mapid="${n}" ${isValid(n) ? "" : "disabled"} style="padding:4px 6px">${labelN}</button>
            <div></div>
            <button data-dir="w" data-mapid="${w}" ${isValid(w) ? "" : "disabled"} style="padding:4px 6px">${labelW}</button>
            <div></div>
            <button data-dir="e" data-mapid="${e}" ${isValid(e) ? "" : "disabled"} style="padding:4px 6px">${labelE}</button>
            <div></div>
            <button data-dir="s" data-mapid="${s}" ${isValid(s) ? "" : "disabled"} style="padding:4px 6px">${labelS}</button>
            <div></div>
        </div>
    `;
    panel.insertAdjacentHTML("beforeend", navHtml);

    panel.querySelectorAll<HTMLButtonElement>("[data-dir]").forEach((btn) => {
        btn.addEventListener("click", () => {
            const id = parseInt(btn.dataset.mapid!, 10);
            if (isValid(id)) onClick(id);
        });
    });
}
```

Click handler : update `currentSelected = neighbourMapId`, call `loadCellGrid(panel, neighbourMapId)` (which already renders cell-grid + we re-call `renderNeighbours` after), and call `reRender(canvas)` to update the white outline on the world canvas.

### Edge case: cross-world neighbour

Si la voisine est dans un autre `worldMap`, son tile n'est pas dans `currentMaps` → `tileByXY[posX,posY]` returns null sur le world canvas, donc l'outline ne s'affiche pas. La cell-grid panel marche quand même (le route `/maps/:mapId` retourne le detail indépendamment du world). Acceptable comme degraded UX. Auto-switch dropdown = future v1.x.

## Build script extension

`build-data.ts` doit extraire les dims de `WorldMapsDataRoot.json`. Le format réel des champs étant incertain (DataCenter dump peut renommer en `m_origineX` ou `origineX` selon la version), utiliser un mapping défensif :

```ts
function extractWorldDims(w: any): WorldMapDims | undefined {
    const origineX = w.origineX ?? w.m_origineX;
    const origineY = w.origineY ?? w.m_origineY;
    const mapWidth = w.mapWidth ?? w.m_mapWidth;
    const mapHeight = w.mapHeight ?? w.m_mapHeight;
    const totalWidth = w.totalWidth ?? w.m_totalWidth;
    const totalHeight = w.totalHeight ?? w.m_totalHeight;
    if ([origineX, origineY, mapWidth, mapHeight, totalWidth, totalHeight].every(Number.isFinite)) {
        return { origineX, origineY, mapWidth, mapHeight, totalWidth, totalHeight };
    }
    return undefined;
}
```

Dans la boucle d'écriture de `areas.json.worlds` :
```ts
worlds: Object.fromEntries(
    dcWorlds.map((w) => {
        const dims = extractWorldDims(w);
        return [w.id, { id: w.id, name: nameOrId(w), ...(dims ? { dims } : {}) }];
    }),
),
```

Si TOUS les worlds ont des dims manquants → world-canvas fallback en colored grid (= v1 behavior). Pas de crash.

## Backend routes — additions

### `GET /api/dofus/cartography/tile/:filename`

Sert un tile JPG static depuis `<dataDir>/cartography/tiles/`.

Validation : regex stricte sur le filename pour éviter path traversal (`../../...`) et forcer le format attendu :
```ts
if (!/^\d{6}_-?\d+\.jpg$/.test(filename)) {
    res.status(400).json({ error: "invalid filename format" });
    return;
}
```

`res.sendFile(path.join(dataDir, "cartography/tiles", filename))`. Express handles ETag/Cache-Control automatiquement → tile chargé 1× par session.

Si fichier manquant → 404. Si dataDir non ready → 503.

### `GET /api/dofus/tile-mapping?world=N`

Retourne juste les tiles pour `world=N` (slice du tile-mapping.json complet) :
```jsonc
{
    "world": 1,
    "tiles": [
        { "index": 0, "name": "1", "scale": "0.2", "address": "0.2/1.jpg",
          "guid": "...", "tile": "000422_1.jpg", "width": 1024, "height": 1024,
          "ambiguous": false },
        ...
    ]
}
```

400 si `world` missing/non-int. 503 si dataReady=false. Si world inconnu → `tiles: []` (pas 404 — le frontend gère le cas vide en tombant sur fallback colored grid).

### `GET /api/dofus/worlds` (modified)

Existing route, juste enrichi : chaque entrée inclut `dims?: WorldMapDims` quand disponible :
```jsonc
{ "worlds": [
    { "id": 1, "name": "Amakna", "mapCount": 6203, "dims": { "origineX": -85, "origineY": -90, "mapWidth": 86, "mapHeight": 43, "totalWidth": 13072, "totalHeight": 5418 } },
    { "id": 2, "name": "World 2", "mapCount": 52 }   // pas de dims → fallback grid
] }
```

## Risques et mitigations

| Risque | Mitigation |
|---|---|
| `WorldMapsDataRoot.json` field naming différent (m_origineX vs origineX) | `extractWorldDims` essaie les 2 nominations. Si ALL dims manquent → `dims: undefined` → fallback colored grid (degraded mais fonctionnel) |
| 112 MB cartography ajoutés au repo (250 MB total avec v1) | Acceptable single-user dev tool. Documenté. v2 : download-on-demand si pénible |
| Tile filenames format inattendu | Regex `/^\d{6}_-?\d+\.jpg$/` accepte les nom avec moins négatif. Si manifest.json pointe sur un fichier hors format → route 400, frontend tombe sur le fillRect placeholder gris (port world.ts:670) |
| `loadTileImage` cache module-level grow infini si user switch beaucoup de worlds | ~1k tiles max sur 35 worlds × ~30 tiles → <100 MB RAM. Acceptable v1.1. v2 : LRU 200 entrées |
| Race condition switch world rapide (tile T1 du world A chargée pendant qu'on render world B) | `worldAtStart = currentWorld` capture dans `loadAndRender`. Avant chaque `await loadTileImage` check `currentWorld !== worldAtStart` → bail. Pattern déjà appliqué dans v1 fix |
| Atlas rendering async progressif (user voit canvas se remplir tile par tile) | UX acceptable (style Google Maps). Order déterministe via `index` du tile-mapping. v2 : fillRect placeholder grey avant chaque tile pour éviter le canvas blanc au démarrage |
| Express sendFile pour 1024×1024 JPGs | ETag/Cache-Control auto. Browser cache → 1 GET par tile par session |
| Cross-world neighbours (clicking N from a map at the world boundary may target another world) | Detail panel works (route fetches by mapId regardless). World canvas outline disappears (acceptable degraded UX). Future v1.x : auto-switch dropdown |
| `MappedTile.scale === "0.2"` filtering hardcodé | Scale 0.2 est le seul qu'on supporte v1.1. Si une world n'a pas de scale 0.2 (rare) → tiles vides → fallback colored grid via the `tiles.length > 0` check |
