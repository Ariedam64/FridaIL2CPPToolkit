# Dofus Plugin — Map Feature v1.2 (Tile-scale fixes + Viewport zoom/pan) — Design Spec

> Iteration v1.2 du plugin Dofus map. Trois fixes après testing manuel de v1.1 : (1) layout des tiles bug — atlas Amakna mal placé (single row au lieu de grille 2×2), (2) scale hardcodé à `"0.2"` exclut les 34 autres mondes (qui utilisent `"0.25"`), (3) viewport DOM fixe 60vh avec zoom-pan local style Google Maps. Sub-project sur la même branche `feat/v1.5-dofus-map`.

**Date** : 2026-05-08
**Branche cible** : `feat/v1.5-dofus-map` (déjà créée pour v1 + v1.1)
**Auteur** : brainstormé avec l'utilisateur après testing manuel de v1.1
**Dépendances** : Dofus Map Feature v1.1 (commits `130209a4..7cda8b45` sur cette branche)

---

## TL;DR

User a smoke-testé v1.1. Trois problèmes :
- **Bug 1 — Layout cassé** : sur Amakna l'atlas s'affiche avec les 4 tiles côte à côte horizontalement (single row) au lieu d'une grille 2×2. Cause : `renderAtlas` wrap sur `wm.totalWidth` (= 10000 world-px) au lieu de `tilesAtlasWidth = totalWidth × tileScale` (= 2000 tile-px à scale 0.2).
- **Bug 2 — Autres mondes vides** : les 34 autres mondes (Wabbit, Frigost, etc.) n'affichent rien. Cause : mon filtre `t.scale === "0.2"` est hardcodé. Inspection de tile-mapping.json : seul Amakna a scale 0.2 ; les autres mondes utilisent `"0.25"` comme scale le plus petit.
- **Bug 3 — Viewport non-isolé** : la zone de la carte étend toute la page web. L'utilisateur veut une zone fixe (60vh) avec son propre zoom-pan interne (style Google Maps : wheel + drag).

v1.2 corrige ces 3 points. Pure dette technique sur v1.1 — pas de nouvelle feature de gameplay.

**Out of scope v1.2** : pinch-zoom touchscreen, double-click zoom, mini-map overview, smooth zoom animation, switching dynamiquement de scale en fonction du niveau de zoom (toujours scale le plus bas).

## Fix #1 + #2 : tile layout & scale dynamique (renderAtlas)

### Bug actuel (v1.1)

```ts
// world-canvas.ts:153
const tilesForScale = opts.tiles.filter((t) => t.scale === "0.2");

// :160
let cursorX = 0, cursorY = 0, rowMaxH = 0;
for (const t of tilesForScale) {
    if (cursorX + t.width > wm.totalWidth + 8) {  // wm.totalWidth = 10000 → wrap NEVER
        cursorY += rowMaxH; cursorX = 0; rowMaxH = 0;
    }
    ...
}
```

Pour Amakna (`wm.totalWidth = 10000`, 4 tiles dont total `1024+976+1024+976 = 4000`), 4000 < 10000+8 → wrap ne déclenche jamais → 4 tiles côte à côte.

Pour les 34 autres mondes : `tilesForScale = []` (filtrent sur 0.2 mais ils ont 0.25, 0.5, ...) → boucle ne s'exécute pas → canvas reste noir.

### Fix combiné

```ts
// 1. Pick le scale le plus bas dynamiquement (plus petit = couverture totale en moins de tiles)
const allScales = [...new Set(opts.tiles.map((t) => parseFloat(t.scale)))].sort((a, b) => a - b);
if (allScales.length === 0) {
    // Fallback : pas de tiles parseables → render placeholder bg + map markers seulement
    // (treat as colored-grid via early return)
    return renderColoredGrid(canvas, ...);
}
const tileScale = allScales[0];  // 0.2 pour Amakna, 0.25 pour les autres
const tilesForScale = opts.tiles.filter((t) => parseFloat(t.scale) === tileScale);

// 2. Compute atlas dimensions in tile-pixels
const tilesAtlasW = wm.totalWidth * tileScale;
const tilesAtlasH = wm.totalHeight * tileScale;

// 3. Canvas natif = tile-pixel dimensions (capped by ATLAS_MAX_W if needed)
const renderScale = Math.min(1, ATLAS_MAX_W / tilesAtlasW);
canvas.width  = Math.round(tilesAtlasW * renderScale);
canvas.height = Math.round(tilesAtlasH * renderScale);

// 4. Tile layout: wrap sur tilesAtlasW (en tile-px), pas sur totalWidth (en world-px)
let cursorX = 0, cursorY = 0, rowMaxH = 0;
for (const t of tilesForScale) {
    if (cursorX + t.width > tilesAtlasW + 8) {
        cursorY += rowMaxH; cursorX = 0; rowMaxH = 0;
    }
    const dx = cursorX * renderScale, dy = cursorY * renderScale;
    const dw = t.width * renderScale, dh = t.height * renderScale;
    // ... drawImage / fillRect placeholder ...
    cursorX += t.width;
    if (t.height > rowMaxH) rowMaxH = t.height;
}

// 5. Map markers: convert world → tile-px → canvas-px
ctx.globalAlpha = 0.35;
for (const m of opts.maps) {
    const a = worldToAtlasXY(wm, m.posX, m.posY);  // returns world-px
    // World-px to tile-px : multiply by tileScale ; then to canvas-px : multiply by renderScale
    const fx = a.x * tileScale * renderScale;
    const fy = a.y * tileScale * renderScale;
    const fw = a.w * tileScale * renderScale;
    const fh = a.h * tileScale * renderScale;
    ctx.fillRect(fx, fy, fw, fh);
    tileByXY[`${m.posX},${m.posY}`] = m.mapId;
}

// 6. Hit-test inverse : canvas-px → tile-px (divide by renderScale) → world-px (divide by tileScale) → world coords
return {
    hitTest: (px, py) => {
        const ax = (px / renderScale) / tileScale;
        const ay = (py / renderScale) / tileScale;
        const { posX, posY } = atlasXYToWorld(wm, ax, ay);
        return tileByXY[`${posX},${posY}`] ?? null;
    },
};
```

### Sanity-check Amakna avec le fix

- `wm.totalWidth = 10000`, `wm.totalHeight = 8000`
- `tileScale = 0.2`, donc `tilesAtlasW = 2000`, `tilesAtlasH = 1600`
- `renderScale = min(1, 3000/2000) = 1` → canvas 2000×1600 (1:1 tile→canvas)
- Tile 1 (1024×1024) à `(0, 0)` ; cursorX=1024, rowMaxH=1024
- Tile 2 (976×1024) à `(1024, 0)` ; cursorX=2000 ; tile 2 fits car `2000 ≤ 2000+8`
- Tile 3 (1024×576) : `2000 + 1024 = 3024 > 2008` → wrap ; cursorY=1024, cursorX=0 → draw à `(0, 1024)` ; cursorX=1024, rowMaxH=576
- Tile 4 (976×576) : `1024 + 976 = 2000 ≤ 2008` → no wrap ; draw à `(1024, 1024)` ; cursorX=2000

Grille 2×2 correcte ✓

### Sanity-check World 2 (Wabbit Island) avec le fix

- Si `tileScale = 0.25` et tile-mapping a 10 tiles en 0.25
- `tilesAtlasW = totalWidth × 0.25` → render normalement avec wrap quand cursorX dépasse cette largeur
- Grille calculée automatiquement, plus de canvas noir

## Fix #3 : Viewport fixe + zoom-pan local

### Layout DOM actuel (v1.1)

```html
<div data-testid="canvas-host"
     style="flex:1;overflow:auto;padding:12px;background:#0a0a0a;display:flex;justify-content:center;align-items:flex-start">
    <canvas data-testid="world-canvas"
            style="image-rendering:pixelated;cursor:crosshair"></canvas>
</div>
```

Le canvas fait sa taille naturelle (atlas 2000×1600 ou plus). Le host scroll verticalement et horizontalement. C'est ce qui fait que la zone "explose" quand le canvas est gros.

### Layout DOM v1.2

```html
<div data-testid="canvas-host"
     style="height:60vh;overflow:hidden;background:#0a0a0a;position:relative;cursor:grab">
    <div data-testid="canvas-viewport"
         style="position:absolute;top:0;left:0;transform:translate(${panX}px,${panY}px) scale(${zoom});transform-origin:0 0;will-change:transform">
        <canvas data-testid="world-canvas"
                style="image-rendering:pixelated;display:block"></canvas>
    </div>
</div>
```

- **Host** : 60vh fixe, hidden overflow → la zone de la map ne grandit jamais peu importe la taille du canvas
- **Viewport** : div interne qui porte le `transform` (translate + scale)
- **Canvas** : taille naturelle inchangée ; le visible est contrôlé par le transform CSS

`transform: translate(...) scale(...)` est l'ordre où :
- D'abord scale autour de transform-origin (= top-left du viewport)
- Ensuite translate (en CSS-pixels du host)

`will-change: transform` indique au browser de réserver une layer GPU → smooth zoom/pan.

### State module-level

```ts
let zoom = 1;            // facteur courant, ∈ [0.25, 4]
let panX = 0;            // px CSS dans le host (left)
let panY = 0;            // px CSS dans le host (top)
const ZOOM_MIN = 0.25;
const ZOOM_MAX = 4;
const ZOOM_STEP = 1.1;   // wheel up = ×1.1, wheel down = ÷1.1
const DRAG_THRESHOLD_PX = 5;
```

### Wheel handler (zoom centré sur curseur)

```ts
host.addEventListener("wheel", (e) => {
    e.preventDefault();
    const rect = host.getBoundingClientRect();
    const mx = e.clientX - rect.left;  // mouse pos in host CSS-px
    const my = e.clientY - rect.top;
    const zoomDelta = e.deltaY < 0 ? ZOOM_STEP : 1 / ZOOM_STEP;
    const newZoom = Math.max(ZOOM_MIN, Math.min(ZOOM_MAX, zoom * zoomDelta));
    if (newZoom === zoom) return;
    // To keep the point under the mouse fixed:
    //   canvasPoint = (mouse - pan) / zoom  (constant before/after)
    //   newPan = mouse - canvasPoint * newZoom
    panX = mx - ((mx - panX) / zoom) * newZoom;
    panY = my - ((my - panY) / zoom) * newZoom;
    zoom = newZoom;
    clampPan(host);
    applyTransform(viewport);
}, { passive: false });
```

### Drag handler (pan vs click)

```ts
let dragging = false;
let dragStart: { x: number; y: number; panX: number; panY: number } | null = null;
let dragMoved = false;

host.addEventListener("mousedown", (e) => {
    if (e.button !== 0) return;  // left button only
    dragStart = { x: e.clientX, y: e.clientY, panX, panY };
    dragMoved = false;
    dragging = true;
    host.style.cursor = "grabbing";
});

window.addEventListener("mousemove", (e) => {
    if (!dragging || !dragStart) return;
    const dx = e.clientX - dragStart.x;
    const dy = e.clientY - dragStart.y;
    if (!dragMoved && Math.hypot(dx, dy) > DRAG_THRESHOLD_PX) {
        dragMoved = true;
    }
    if (dragMoved) {
        panX = dragStart.panX + dx;
        panY = dragStart.panY + dy;
        clampPan(host);
        applyTransform(viewport);
    } else {
        // Still in click territory — don't pan, leave hover hit-test alone
    }
});

window.addEventListener("mouseup", (e) => {
    if (!dragging) return;
    dragging = false;
    host.style.cursor = "grab";
    if (!dragMoved && currentHitTest && dragStart) {
        // It was a click. Use the original mousedown coords (e.clientX may differ slightly
        // due to micro-movements). dragStart.x/y are screen coords ; convert to canvas-coords.
        const rect = host.getBoundingClientRect();
        const mx = dragStart.x - rect.left;
        const my = dragStart.y - rect.top;
        const canvasX = (mx - panX) / zoom;
        const canvasY = (my - panY) / zoom;
        const hit = currentHitTest.hitTest(canvasX, canvasY);
        if (hit !== null) {
            currentSelected = hit;
            void reRender(canvas);
            void loadCellGrid(panel, canvas, hit);
        }
    }
    dragStart = null;
});
```

`mousemove` et `mouseup` sont attachés à `window` (pas au host) pour capturer les mouvements même si la souris sort temporairement du host pendant un drag. Pour v1.2 on accepte que ces listeners restent jusqu'à ce que la page soit déchargée — `mountMap` est appelé une seule fois quand l'utilisateur ouvre l'onglet Map et reste mounted pendant toute la session du plugin (pas de SPA navigation back). Cleanup propre via `AbortController` est listé comme out-of-scope (v1.3).

### Hover hit-test

Le mousemove sur `host` (pour update `currentHover`) doit aussi transformer les coords :

```ts
host.addEventListener("mousemove", (e) => {
    if (dragging) return;  // pan mode: skip hover update
    if (!currentHitTest) return;
    const rect = host.getBoundingClientRect();
    const canvasX = ((e.clientX - rect.left) - panX) / zoom;
    const canvasY = ((e.clientY - rect.top) - panY) / zoom;
    const hit = currentHitTest.hitTest(canvasX, canvasY);
    if (hit !== currentHover) {
        currentHover = hit;
        void reRender(canvas);
        // ... hoverInfo update ...
    }
});
```

### `clampPan`

Empêche le canvas de quitter complètement le viewport. Au minimum :

```ts
function clampPan(host: HTMLElement, canvas: HTMLCanvasElement): void {
    const hostW = host.clientWidth;
    const hostH = host.clientHeight;
    const canvasW = canvas.width * zoom;
    const canvasH = canvas.height * zoom;
    if (canvasW <= hostW) {
        panX = (hostW - canvasW) / 2;     // canvas smaller than host → center horizontally
    } else {
        panX = Math.max(hostW - canvasW, Math.min(0, panX));  // bounded so canvas stays in viewport
    }
    if (canvasH <= hostH) {
        panY = (hostH - canvasH) / 2;     // center vertically
    } else {
        panY = Math.max(hostH - canvasH, Math.min(0, panY));
    }
}
```

### Reset zoom/pan on world switch

Dans le handler de `select.addEventListener("change", ...)` :

```ts
select.addEventListener("change", () => {

    currentSelected = null;
    currentHover = null;
    zoom = 1;       // NEW: reset
    panX = 0;       // NEW: reset
    panY = 0;       // NEW: reset
    // ... existing logic ...

```

(Le `applyTransform` se fait au prochain `loadAndRender → reRender → applyTransform`. Ou explicitement après reset.)

### `applyTransform` helper

```ts
function applyTransform(viewport: HTMLElement): void {
    viewport.style.transform = `translate(${panX}px, ${panY}px) scale(${zoom})`;
}
```

Appelé après chaque modif de zoom/panX/panY. Aussi appelé après chaque `reRender` pour fitter au cas où le canvas a changé de taille (ex: switch world).

### Initial fit on load

Quand un monde est rendu, on veut que la map soit visible entièrement par défaut (zoom=fit, pan=center). Pas systématiquement zoom=1 :

```ts
function fitToHost(host: HTMLElement, canvas: HTMLCanvasElement): void {
    const hostW = host.clientWidth;
    const hostH = host.clientHeight;
    if (canvas.width === 0 || canvas.height === 0) return;
    zoom = Math.min(hostW / canvas.width, hostH / canvas.height, 1);
    panX = (hostW - canvas.width * zoom) / 2;
    panY = (hostH - canvas.height * zoom) / 2;
}
```

Appelé à la fin de `loadAndRender` (après le canvas a sa nouvelle taille).

## Risques et mitigations

| Risque | Mitigation |
|---|---|
| `parseFloat(t.scale)` retourne NaN sur des données malformées | `allScales` filter out NaN avant sort |
| Fallback `renderColoredGrid` quand allScales est vide | Cas extrême où le manifest a tile-mapping pour ce world mais aucune tile parseable. Cas rare mais documenté |
| Zoom au-delà des limites due à wheelDelta très grand | `Math.max(ZOOM_MIN, Math.min(ZOOM_MAX, ...))` borne dans tous les cas |
| Drag start hors host puis end dans host (inverse aussi) | mousemove/mouseup attachés à window pour robustesse |
| Memory leak : window listeners pas cleanup quand mountMap re-mount | Pour v1.2 acceptable (single page, no SPA navigation back). v1.3 : `AbortController` + cleanup function |
| `clampPan` math incorrecte | Plusieurs sanity-checks (canvas plus petit que host = center ; plus grand = pan dans bornes) |
| User dezoome au max → tiny atlas perdu au centre | clampPan avec mode "fit" force centrage si canvas plus petit que host |
| Click qui déclenche un fetch alors que l'user voulait pan (drag < 5px par accident) | Threshold 5px est un compromis ; trop petit = false-pan, trop grand = "clicks ignorés". 5 est standard |
| `transform-origin: 0 0` + `translate(...) scale(...)` est l'ordre correct | Vérifier en code et en test |
